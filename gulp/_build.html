    # 项目分为两类：

    传统未模块化的项目；使用seaJS,requireJS等模块化的项目

    # 以requireJS模块化项目为例：

    # 1，解析所有非第三方文件的依赖文件路径，并存入对象a，a大概如下：

    a = {
   "./main.js": ['./common/a.js', './lib/b.js'],
   "./page/page1.js": ['./common/a.js', './lib/b.js'，‘./hh.js’]
   ...
}

    # 2，首先把所有main.js依赖的文件包括main自身打包成一个文件，并将所以已被打包文件存入数组b，b大概如下：

    b = [
       "./main.js",
       "./common/a.js",
       './lib/b.js'
        ...
]

    # 3, 遍历所有单个页面模块js，提取出所有公共资源（至少两个模块引用了这个资源），并且去掉已经存在于b数组的资源。将这些资源放置于对象c

    比如page1.js和page2.js都引用了"'./common/a.js"和"'./common/b.js",但是"'./common/a.js"已经存
    在b中，就只将"'./common/b.js"放入c；c大致如下：

    c = {
       "./page/page1.js": [./common/b.js],
       "./page/page2.js": [./common/b.js],
       ...
}

    将所有列入c的value数组中的文件push入b
    再将c(key->value)中value包含的所有文件打包成一个公共文件，并修改key文件的引用路径

    # 4，再重复遍历所有单个页面模块js，将所有被该模块引用的但不存在b中的文件打包成单个文件并修改引用路径；

    比如"./page/page3.js"中引入 ./common/b.js 和 ./hh.js 和 test.js ,则把后两者打包成一个公共js并修
    改"./page/page3.js"中的引用路径

    # 加版本号

    在上面的压缩合并文件后，就可以使用md5工具对压缩合并后的文件进行hash处理，再修改引用
    比如将 ./hh.js 和 test.js 打包成 th.min.js 后hash化处理成 th_2213332dfas.min.js, 再修改
    ./page/page3.js 中的引用路径

    # 非模块化的项目

    非模块化的项目一般不会有文件的重复引用，只需要直接将当前页面所有引入资源合并压缩成单个资
    源即可； 如果的确有多页面之间引用了相同的文件，那么同样可以使用gulp的through工具遍历文件提取公共文
    件进行压缩合并
