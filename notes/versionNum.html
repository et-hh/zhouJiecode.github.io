    # 利用框架本身的资源加载器

    举个栗子：requireJS requireJS有个配置参数：urlArgs，在源码中有这么一段：

    return config.urlArgs ? url +
          ((url.indexOf("?") === -1 ? '?' : '&') +
          config.urlArgs) : url;

    源码中只有这里用到了urlArgs； 从代码分析来看，其实就是通过js创造了一个script标签然后设置url为以上返回值，而上面url后面加上了一个版本号参数：urlArgs，从而给要加载的js加上了版本号。

    其他框架也大都能自己加载前端资源，那么方法差不多也是类似的

    其实还有种方式：就是gulp等编译工具和框架相结合，这种需要修改源码，这里有篇文章就是描述这个的：http://www.cnblogs.com/shihao905/p/6229644.html

    # 使用js手动加载资源

    这种方式有两个分支：

    # （1）：使用js创造<script>标签，然后动态添加到dom中，比如：

                     var head = document.getElementsByTagName('head')[0];
                 var script = document.creatElement('script');

                 script.defer = 'defer';
                 script.async = false;
                 script.type = "text/javascript";
                 script.src = 'XX?v=' + version;
                 head.insertBefore(script, head.firstChild);

    这种方式会引起页面重绘，不适合刚进页面时使用，否则会影响页面加载性能

    # (2):使用document.write,如：

    document.write('<script type="text/javascript" src="XX?v='+ version +'"></script>')

    由于document.write在onload事件之后再使用会直接重写页面，所以这个只能在页面刚进入时使用
