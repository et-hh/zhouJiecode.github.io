    # 将代码中使用的一些npm包从网上公共cdn引

    目的：减轻自己的服务器的访问压力

    # 图片尽量使用jpg格式，并且放tinypng上进行压缩

    目的：减少代码量（webpack打包后小图片会直接通过数据流方式写死在代码里），减轻服务器访问压力（大图片还是通过网络请求方式获取）

    # 第三方库文件打成一个包，自己写的公共文件打成一个包

    目的：

    • 第三方库基本不会改动，通过webpack.ensure生成的单独chunk在客户端只需加载一次，后续无论自己代码怎么改，因为这个chunk的文件名没变，浏览器都会直接使用缓存，不需要重新加载第三方库文件
    • 自己写的公共文件相对来说也会改动较少，目的同上

    # 按需加载

    • 路由动态引用：component：resolve => require(['../a'],resolve)
    • 首页不使用的第三方库通过webpack.ensure([], (require)=>{var a=require('a');...})方式在子页面动态引入，而非直接打包在首页中（这个技术叫webpack code splitting）

    目的：

    • 第三方库基本不会改动，目的同上
    • 首页加载代码量大大减少，因为类似echarts、ztree等第三方库代码量较大，若是直接打包在首页的vendor.js里，首页加载代码量将增大很多，而且这些代码本身首页就不需要

    # 小图标尽量使用字体图标或者svg矢量图

    页面上的一些需要hover变色的小图标，不要使用多张png图片，而是使用字体图标或者svg矢量图，从而减少代码量 ui给我们psd时，让ui直接使用illustrator将其中的小图标做成svg提供给我们，我们这边可以使用svgo对svg进行压缩处理，然后引入代码
